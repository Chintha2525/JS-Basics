JavaScript Promises

  • "Producing code" is code that can take some time
  • "Consuming code" is code that must wait for the result
  • A Promise is an Object that links Producing code and Consuming code

JavaScript Async/Await

  • "async and await make promises easier to write"
  • async makes a function return a Promise
  • await makes a function wait for a Promise
  • // Function to double a number after a 1-second delay
    async function getDoubleAfterDelay(number) {
    return new Promise((resolve) => {
    setTimeout(() => {
    resolve(number * 2);
    }, 1000);
    });
    }

    // Function to calculate the sum of three doubled numbers
    async function calculateSum() {
    const result1 = await getDoubleAfterDelay(5);
    const result2 = await getDoubleAfterDelay(10);
    const result3 = await getDoubleAfterDelay(15);

    const sum = result1 + result2 + result3;
    return sum;
    }

    // Example usage
    calculateSum().then((sum) => {
    console.log("Sum:", sum); // Output: Sum: 60
    });

Basics of REACT

What is REACT?

  • React is a JavaScript library for building user interfaces.
  • React is used to build single-page applications.
  • React allows us to create reusable UI components.

Components

  • A component in React is a reusable, independent piece of UI that can be used to build complex interfaces.
  • Components allow you to split the UI into smaller, manageable parts, making development more efficient.
  • React has two main types of components
    1. Functional Components (Recommended)
    2. Class Components (Older Approach)
